eyhn.page 让你在子域名上托管静态站点。选一个前缀，带上自己的 site key， 用一条 curl 就能发布——不需要构建流水线，也不需要信用卡。

https:// your-name .eyhn.page/

⚡ 秒级发布

上传未压缩的 tar 快照，整站替换，立刻可访问。

🔑 自带密钥

Site key 由你保管，至少 16 字节。丢了就无法更新或删除。

🧹 自动清理

可设置 expiresAt，到期后前缀释放，别人可以重新认领。

三步上线

  1. 构建静态文件 把 dist/ 或任意目录打成纯 .tar（不要 .tar.gz）。
  2. 认领前缀 前缀只能是 a-z、0-9、连字符，且不能以连字符开头或结尾。
  3. PUT 发布 带上 X-Site-Key，整个站点快照一次性替换。

发布命令

# package
COPYFILE_DISABLE=1 tar --no-xattrs --no-mac-metadata -cf site.tar -C dist .

# publish
curl -X PUT "https://eyhn.page/api/sites/demo/publish" \
  -H "X-Site-Key: your-site-key-here" \
  -H "Content-Type: application/x-tar" \
  --data-binary @site.tar

小规则，大省心

📄 index.html

目录请求会找 index.html；/about 也能直接映射 about.html。

🚫 404.html

存在时，缺失路径返回该文件，HTTP 状态码仍是 404。

📦 1 GB 上限

默认最大 tar 体积 1024 MB，足够大多数演示站点。
